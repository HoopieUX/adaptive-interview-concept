# Walkthrough

The full step-by-step version of both flows summarised in the [README](README.md).

## Candidate flow

1. **Landing page.** Three options: try the live chat, watch example conversations, or view
   example reports.
2. **Start the chat.** It opens automatically with a question — no blank page, no "click to
   start."
3. **Answer however comes naturally.** Type or dictate with the mic button. Half-formed thoughts
   are genuinely fine.
4. **The bot adapts.** If an answer is vague, it pushes for a concrete example before moving on.
   If it's already concrete, the bot either moves toward the next trait or tests whether the same
   insight holds in a different, unexplored situation — that's the harder test, and the one that
   actually distinguishes real judgment from a rehearsed answer.
5. **The bot decides when it has enough signal**, typically after 8-12 exchanges, hard-capped so
   it can never run long. The candidate can also ask to wrap up early at any point.
6. **The report is generated the moment the chat ends.** No separate submit step — the finished
   transcript is scored automatically.

## Hiring-manager flow

1. **Example reports list.** Four example candidates, sorted by Cultural Fit %, each shown as a
   card: avatar, a one-line summary, a coloured pill per trait (red, amber, or green), and a
   "X strong / Y flagged" count.
2. **Open one candidate.** The report leads with a TL;DR banner: a verdict (strong, mixed, or weak
   cultural fit) and a two-column quick-scan of which traits stood out as strengths and which are
   worth probing further.
3. **Per-trait detail.** Score, rationale, and the specific quote from the transcript the score is
   based on — never a bare number with no explanation.
4. **Suggested follow-up.** Where a trait's evidence was thin, a specific question is suggested for
   the live interview — built to test whether the trait actually transfers to a new situation, not
   just to re-ask the same thing a different way.
5. **Interview Guide — general reference.** The same for every candidate: what each trait sounds
   like, weak through partial through strong, so an interviewer can reuse it for anyone, not just
   this one candidate.
6. **Full transcript**, available underneath for anyone who wants to see exactly how the
   conversation actually went, not just the summary of it.
